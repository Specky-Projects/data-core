1 Visão Geral da Arquitetura

O sistema de coleta de produtos do Poupi é composto por quatro camadas que se comunicam sequencialmente:

collection_targets
(banco de dados)
→
EcommerceURLScraper
(coletor)
→
raw_collections
(dados brutos)
→
normalized_products
(dados limpos)
→
product_price_analytics
(análises)

Componentes-chave

Componente Arquivo Função
collection_targets database/models.py Tabela de URLs que serão coletadas
EcommerceURLScraper collectors/ecommerce/url_scraper.py Scraper com suporte a VTEX e JSON-LD
BaseCollector collectors/base.py Contrato base para todos os coletores
CollectorRegistry collectors/registry.py Mapa de nome → classe do coletor
DEFAULT_COLLECTION_TARGETS scheduler/jobs.py URLs padrão semeadas no banco na inicialização
run_ecommerce_url_targets_job scheduler/jobs.py Job agendado que dispara o scraper a cada 2h

Estratégias de Scraping

O EcommerceURLScraper tenta as estratégias nesta ordem:

  1. VTEX Catalog API — extrai o productId da URL e chama a API interna da VTEX (/api/catalog_system/pub/products/search?fq=productId:...). Funciona em qualquer loja que use o CMS VTEX.
  2. JSON-LD Structured Data — faz download do HTML e procura por <script type="application/ld+json"> com @type: "Product". Fallback genérico para qualquer site.
  3. Falha documentada — salva {"success": false, ...} para auditoria.

2 Verificar se o Site é VTEX

Antes de adicionar qualquer URL, verifique se a loja usa VTEX. Se usar, o scraper já funciona automaticamente — não é necessário escrever código.

Como identificar uma loja VTEX

  • Abra a página do produto e procure a URL. Se terminar em -<números>.html ou /p/<números>, provavelmente é VTEX.
  • Abra o DevTools (F12) → aba Network → recarregue → procure chamadas para /api/catalog_system/pub/products/search. Se existir, é VTEX.
  • Veja o código-fonte da página (Ctrl+U) e procure por vtex, vtex.com ou VTEX_STATE.

Lojas VTEX já suportadas pelo scraper:

drogasil drogaraia pague menos nissei ultrafarma panvel kabum centauro qualquer loja VTEX
✅

Se o site for VTEX: pule para a Seção 3 e adicione a URL diretamente. Nenhum código novo é necessário.

ℹ️

Se o site não for VTEX: o scraper vai tentar JSON-LD automaticamente. Se também não funcionar, veja a Seção 6 para criar um coletor customizado.

3 Adicionar URL via Banco de Dados

Este é o método mais rápido. A URL entra no banco de dados e o scheduler a coleta automaticamente na próxima execução (a cada 2 horas).

SQL direto

Execute no PostgreSQL (use psql, DBeaver, pgAdmin, etc.):

INSERT INTO collection_targets (
    id,
    module,
    source_name,
    collector_name,
    target_url,
    active,
    metadata_json,
    created_at,
    updated_at
) VALUES (
    gen_random_uuid(),
    'ecommerce',            -- sempre 'ecommerce' para produtos
    'drogasil',             -- nome da loja (snake_case)
    'ecommerce.url_scraper', -- sempre este valor
    'https://www.drogasil.com.br/fralda-pampers-premium-care-m-x44-1351898.html',
    true,                   -- true = ativo, false = pausado
    '{"kind": "product", "owner": "poupi", "category": "baby", "product_seed": "pampers premium care M"}',
    now(),
    now()
);

Campos obrigatórios

Campo Valor Descrição
module 'ecommerce' Sempre este valor para produtos
source_name 'nome_da_loja' Nome da loja em snake_case (ex: pague_menos)
collector_name 'ecommerce.url_scraper' Sempre este valor
target_url URL completa URL da página do produto (com https://)
active true true = coleta ativa, false = pausada
metadata_json JSON Veja campos do metadata abaixo

Campos do metadata_json

Campo Exemplo Descrição
kind "product" Tipo do target (sempre "product" para URLs de produto)
owner "poupi" Sistema ou cliente dono do monitoramento
category "baby" Categoria do produto
product_seed "pampers premium care M" Nome descritivo para identificação humana

Adicionar múltiplos URLs de uma vez

INSERT INTO collection_targets
  (id, module, source_name, collector_name, target_url, active, metadata_json, created_at, updated_at)
VALUES
  (gen_random_uuid(), 'ecommerce', 'drogasil',    'ecommerce.url_scraper',
   'https://www.drogasil.com.br/fralda-pampers-xxg-1351899.html',
   true, '{"kind":"product","owner":"poupi","category":"baby","product_seed":"pampers XXG"}',
   now(), now()),

  (gen_random_uuid(), 'ecommerce', 'drogaraia',   'ecommerce.url_scraper',
   'https://www.drogaraia.com.br/fralda-pampers-xxg-5001234.html',
   true, '{"kind":"product","owner":"poupi","category":"baby","product_seed":"pampers XXG"}',
   now(), now());
⚡

Coleta imediata: após inserir, o scheduler coleta na próxima execução (até 2h). Para testar agora, use o endpoint da API:

POST /api/v1/pipeline/jobs/run-ecommerce-url-targets

Desativar uma URL temporariamente

UPDATE collection_targets
SET active = false, updated_at = now()
WHERE target_url = 'https://...';

4 Adicionar nos Targets Padrão do Código

Se você quer que a URL seja semeada automaticamente sempre que o sistema inicializar (útil para produtos essenciais do Poupi), adicione em scheduler/jobs.py na lista DEFAULT_COLLECTION_TARGETS.

# scheduler/jobs.py

DEFAULT_COLLECTION_TARGETS: list[dict] = [
    # ... targets existentes ...

    # Adicione aqui no final:
    {
        "module":         "ecommerce",
        "source_name":    "minha_nova_loja",
        "collector_name": "ecommerce.url_scraper",
        "target_url":     "https://www.minhaloja.com.br/produto-123.html",
        "active":         True,
        "metadata_json": {
            "kind":         "product",
            "owner":        "poupi",
            "category":     "baby",
            "product_seed": "nome do produto para referência",
        },
    },
]
⚠️

Atenção: A função ensure_default_collection_targets() é idempotente — ela usa INSERT OR IGNORE baseado na chave única (module, source_name, collector_name, target_url). Duplicatas não são inseridas.

Após alterar o código, reinicie o container do scheduler para que a semeadura ocorra:

docker restart data-core-scheduler-1

5 Adicionar Suporte a uma Nova Loja VTEX

O EcommerceURLScraper detecta o domínio da URL e aplica regras específicas por loja. Para adicionar uma nova loja VTEX com comportamento padrão, abra collectors/ecommerce/url_scraper.py e localize o dicionário de lojas suportadas.

Estrutura atual

# collectors/ecommerce/url_scraper.py

VTEX_STORES = {
    "drogasil.com.br":    "drogasil",
    "drogaraia.com.br":   "drogaraia",
    "paguemenos.com.br":  "pague_menos",
    "nissei.com.br":      "nissei",
    "ultrafarma.com.br":  "ultrafarma",
    "panvel.com":         "panvel",
    # ↓ Adicione aqui
    "novafarma.com.br":   "nova_farma",
}

Adicione a entrada com o domínio da loja (sem https://www.) mapeado para um store_name em snake_case. Isso é suficiente para que o scraper identifique a loja e tente a API VTEX automaticamente.

🔍

Teste rápido antes de adicionar: verifique se a API VTEX funciona manualmente abrindo no navegador:

https://www.novafarma.com.br/api/catalog_system/pub/products/search?fq=productId:123456

Se retornar JSON com dados do produto, a integração VTEX funcionará automaticamente.

6 Criar um Coletor do Zero (Loja Não-VTEX)

Se o site não usa VTEX e não tem JSON-LD estruturado, você precisa criar um BaseCollector customizado com lógica de scraping específica.

Template completo

# collectors/ecommerce/minha_loja_collector.py
from __future__ import annotations
import requests
from collectors.base import BaseCollector, CollectedItem, CollectorMetadata, CollectorDomain


class MinhaLojaCollector(BaseCollector):
    """
    Coletor para minha-loja.com.br.
    Estratégia: HTML scraping com BeautifulSoup.
    """

    # ── Metadados do coletor (obrigatórios) ─────────────────────
    metadata = CollectorMetadata(
        name="ecommerce.minha_loja",         # chave única no registry
        domain=CollectorDomain.ECOMMERCE,
        source="minha_loja",
        description="Coleta preços de minha-loja.com.br",
        default_interval_minutes=120,
        collector_version="1.0.0",
        raw_schema_name="scrapedProduct",
        raw_schema_version="1.0.0",
        schedulable=True,
    )

    # ── URLs de produtos a coletar (ou busque do config/DB) ──────
    PRODUCT_URLS: list[str] = [
        "https://www.minha-loja.com.br/produto-abc",
        "https://www.minha-loja.com.br/produto-xyz",
    ]

    def collect(self) -> list[CollectedItem]:
        """Implementação obrigatória do BaseCollector."""
        items = []
        headers = {"User-Agent": "Mozilla/5.0 (compatible; PouplBot/1.0)"}

        for url in self.PRODUCT_URLS:
            try:
                resp = requests.get(url, headers=headers, timeout=15)
                resp.raise_for_status()

                # Extraia os dados conforme o HTML da loja
                price = self._extract_price(resp.text)
                title = self._extract_title(resp.text)

                payload = {
                    "success":        True,
                    "scrapedProduct": {
                        "title":        title,
                        "price":        price,
                        "currency":     "BRL",
                        "availability": "in_stock",
                        "store_name":   "minha_loja",
                        "url":          url,
                        "scraper_strategy": "custom_html",
                    }
                }
                items.append(CollectedItem(
                    payload=payload,
                    external_id=url,
                    source_url=url,
                    metadata={"store": "minha_loja"},
                ))
            except Exception as e:
                items.append(CollectedItem(
                    payload={"success": False, "error_message": str(e)},
                    external_id=url,
                    source_url=url,
                    metadata={},
                ))

        return items

    def _extract_price(self, html: str) -> float | None:
        # Implemente extração específica da loja
        from bs4 import BeautifulSoup
        soup = BeautifulSoup(html, "html.parser")
        el = soup.select_one(".product-price")  # ajuste o seletor
        if el:
            return float(el.text.strip().replace("R$", "").replace(",", ".").strip())
        return None

    def _extract_title(self, html: str) -> str | None:
        from bs4 import BeautifulSoup
        soup = BeautifulSoup(html, "html.parser")
        el = soup.select_one("h1.product-name")  # ajuste o seletor
        return el.text.strip() if el else None

7 Registrar o Coletor no Registry

Abra collectors/registry.py e adicione seu coletor:

# collectors/registry.py
from collectors.ecommerce.minha_loja_collector import MinhaLojaCollector

# No corpo do CollectorRegistry (ou no dict de inicialização):
_registry: dict[str, type[BaseCollector]] = {
    # ... entradas existentes ...
    MinhaLojaCollector.metadata.name: MinhaLojaCollector,  # ← adicione
}

8 Agendar o Coletor no Scheduler

Se seu coletor é um BaseCollector com schedulable=True, adicione-o ao mapeamento em scheduler/jobs.py:

# scheduler/jobs.py

MODULE_COLLECTORS = {
    "ecommerce": [
        "ecommerce.generic_product",
        "ecommerce.minha_loja",  # ← adicione aqui
    ],
    # ... outros módulos ...
}

Se quiser agendar com horário específico, adicione um job no scheduler/service.py:

# scheduler/service.py
from scheduler.jobs import collect_raw_job

scheduler.add_job(
    collect_raw_job,
    "interval",
    hours=2,
    kwargs={"collector_name": "ecommerce.minha_loja"},
    id="collect_minha_loja",
    replace_existing=True,
)

9 Verificar se Está Funcionando

  1. Verifique se a URL foi inserida no banco 
    SELECT id, source_name, target_url, active, created_at
FROM collection_targets
WHERE module = 'ecommerce'
ORDER BY created_at DESC
LIMIT 10;
  2. Verifique se existem coletas para a URL 
    SELECT rc.id, rc.source_name, rc.processing_status, rc.collected_at,
       rc.raw_json->>'success' as success,
       rc.raw_json->'scrapedProduct'->>'price' as price
FROM raw_collections rc
WHERE rc.source_name = 'minha_loja'
ORDER BY rc.collected_at DESC
LIMIT 5;
  3. Verifique se normalizou corretamente 
    SELECT id, store_name, title, price, availability, collected_at, analytics_status
FROM normalized_products
WHERE store_name = 'minha_loja'
ORDER BY collected_at DESC
LIMIT 5;
  4. Veja o histórico de execuções do job 
    SELECT collector_name, status, items_collected, error_count, started_at, finished_at
FROM collection_runs
WHERE collector_name LIKE '%minha_loja%'
ORDER BY started_at DESC
LIMIT 10;
  5. Dispare uma coleta manual via API 
    curl -X POST https://seu-api/api/v1/pipeline/jobs/run-ecommerce-url-targets \
     -H "X-API-Key: sua-chave"

10 Solução de Problemas

Sintoma Causa Provável Solução
success: false em raw_collections URL inválida, produto fora de estoque, site com anti-bot Verifique error_message no raw_json. Tente abrir a URL no navegador.
Nenhuma linha em raw_collections após esperar URL com active=false ou coletor não registrado Verifique SELECT active FROM collection_targets WHERE target_url=...
scraper_strategy: "failure" Nem VTEX nem JSON-LD encontrados O site não é VTEX e não tem JSON-LD. Crie um coletor customizado (Seção 6).
Produto normalizado mas price é NULL Normalizer não encontrou o campo price no payload Verifique se o scrapedProduct.price está no raw_json.
URL duplicada no banco Constraint única (module, source_name, collector_name, target_url) Normal — o sistema rejeita duplicatas automaticamente.
Coleta travada (status=running por +30 min) Job travou; circuit breaker ativo cleanup_stale_runs_job corrige automaticamente. Ou execute: UPDATE collection_runs SET status='failed' WHERE status='running' AND started_at < now()-interval '30 min'
Erro ImportError ao importar coletor Dependência ausente (ex: beautifulsoup4) Adicione ao requirements.txt e reconstrua a imagem Docker.

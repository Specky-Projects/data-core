1 Core Operacional Core

Tabelas que gerenciam o ciclo de vida das coletas: quais URLs coletar, quais coletores existem, quais execuções ocorreram e quais erros aconteceram.

collection_targets
URLs que serão coletadas pelo sistema de scraping
ColunaTipoDescrição
idUUID PKIdentificador único
moduleString(80)Módulo dono (ecommerce, real_estate…)
source_nameString(160)Nome da loja/fonte (ex: drogasil)
collector_nameString(160)Coletor responsável (ex: ecommerce.url_scraper)
target_urlTextURL completa do produto a ser coletado
activeBooleantrue = coleta ativa; false = pausada
metadata_jsonJSONBDados extras: kind, owner, category, product_seed
created_at / updated_atDateTime TZAuditoria temporal

Restrição única: (module, source_name, collector_name, target_url)

-- Todos os targets ativos do ecommerce
SELECT source_name, target_url, metadata_json->>'product_seed' as produto
FROM collection_targets
WHERE module = 'ecommerce' AND active = true
ORDER BY source_name;

enabledBooleanColetor habilitado no sistema
configJSONBConfigurações do coletor (parâmetros, credenciais referenciadas)
created_at / updated_atDateTime TZAuditoria
collection_runs
Histórico de cada execução de coleta (job run)
ColunaTipoDescrição
idUUID PKIdentificador
collector_idUUID FK → collector_definitionsColetor que rodou
collector_nameString(160)Nome do coletor (desnormalizado)
module / domain / sourceString/EnumContexto do coletor

items_collected / raw_saved_countIntegerItens coletados e salvos

-- Últimas execuções com erros
SELECT collector_name, status, error_count, error_message, started_at
FROM collection_runs
WHERE status IN ('failed', 'partial')
ORDER BY started_at DESC LIMIT 20;

2 Camada Raw Raw

Armazena todos os dados brutos exatamente como chegaram das fontes, antes de qualquer transformação. É a fonte da verdade do sistema.

raw_collections
Tabela central de ingestão — cada linha é uma resposta de coleta
ColunaTipoDescrição
idUUID PKIdentificador único do dado bruto
moduleString(80)Domínio: ecommerce, crypto, real_estate…
source_nameString(160)Loja/exchange/fonte (ex: drogasil, binance)
source_typeString(80)Mecanismo: python_url_scraper, legacy_ts_scraper…
collector_nameString(160)Coletor que gerou o dado
raw_schema_nameString(160)Schema do payload: scrapedProduct, marketCandle…
target_url / urlTextURL de onde o dado foi coletado
raw_jsonJSONBO payload completo da coleta (dado bruto)
checksumString(64)SHA-256 do raw_json (deduplicação)
processing_statusString(40)normalization_pending | normalized | ignored
collected_at / created_atDateTime TZTimestamps da coleta e inserção
error_messageTextErro de coleta (quando success=false)
metadata_jsonJSONBMetadados adicionais (strategy, collection_target_id…)

Restrição única: (module, source_name, checksum) — evita duplicatas automaticamente

-- Ver últimas coletas com preço extraído
SELECT
    source_name,
    raw_json->'scrapedProduct'->>'title'  as titulo,
    raw_json->'scrapedProduct'->>'price'  as preco,
    processing_status,
    collected_at
FROM raw_collections
WHERE module = 'ecommerce'
  AND raw_json->>'success' = 'true'
ORDER BY collected_at DESC LIMIT 10;

raw_schema_name / raw_schema_versionStringSchema que o coletor gera
is_activeBooleanSe esta versão está ativa atualmente
description / metadata_jsonText/JSONBDocumentação da versão

3 Camada Normalizada Normalized

Dados transformados, limpos e estruturados. Cada tabela corresponde a um tipo de entidade de negócio. São alimentadas pelos normalizers que leem o raw_json e extraem campos tipados.

canonical_product_idString(255)ID canônico para agrupar o mesmo produto entre lojas
source_urlTextURL da página do produto (link para o consumidor)
title / brandText / String(160)Nome e marca do produto
price / currencyNumeric(14,2) / String(8)Preço e moeda (BRL)
availabilityString(80)in_stock | out_of_stock
store_nameString(160)Loja (ex: drogasil, drogaraia)
analytics_statusString(40)pending | processed

-- Comparar preço do mesmo produto em várias lojas
SELECT store_name, price, availability, collected_at
FROM normalized_products
WHERE canonical_product_id = 'pampers-premium-care-m'
  AND collected_at >= now() - interval '24 hours'
ORDER BY price ASC;

4 Camada Analytics Analytics

Métricas calculadas e insights derivados dos dados normalizados. São alimentadas pelos analytics processors.

product_price_analytics
Análise de preços de produtos (Poupi)
ColunaTipoDescrição
idUUID PK
product_idUUID FK → normalized_productsProduto analisado
avg_price_7d / avg_price_30dNumeric(14,2)Preço médio nos últimos 7 e 30 dias
min_price_90d / max_price_90dNumeric(14,2)Mínimo e máximo nos últimos 90 dias
price_scoreNumeric(10,4)Score de oportunidade de preço (0-100)
calculated_atDateTime TZQuando a análise foi calculada
-- Produtos com melhor oportunidade de preço agora
SELECT np.store_name, np.title, np.price,
       pa.avg_price_30d, pa.min_price_90d, pa.price_score
FROM product_price_analytics pa
JOIN normalized_products np ON pa.product_id = np.id
WHERE pa.price_score >= 80
ORDER BY pa.price_score DESC LIMIT 20;

5 Observabilidade do Pipeline Pipeline

Instrumentação de todas as execuções do pipeline para monitoramento operacional e debugging.

pipeline_runs
Log de execuções de stages do pipeline (coleta, normalização, analytics)
ColunaTipoDescrição
idUUID PK
domainString(64)crypto | ecommerce | real_estate | sports_betting | trading
stageString(32)collection | normalization | analytics
source_nameString(128)Classe que executou (ex: nome do normalizer)
started_at / finished_atDateTime TZInício e fim
duration_secondsFloatDuração em segundos
statusString(16)running | success | partial | error
items_input / processed / skipped / errorIntegerContadores de itens por resultado
triggerString(32)scheduler | api | manual
-- Resumo de execuções das últimas 24h
SELECT domain, stage, status, count(*) qtd,
       sum(items_processed) total_itens
FROM pipeline_runs
WHERE started_at >= now() - interval '24 hours'
GROUP BY 1,2,3 ORDER BY 1,2,3;

7 Governança de Dados Governance

Tabelas de documentação e rastreabilidade do sistema de dados.

data_lineage
Rastreamento completo raw → normalized → analytics

Cada linha conecta um raw_collection_id ao registro normalizado e analítico correspondente. Permite rastrear de onde veio qualquer dado.

ColunaTipoDescrição
raw_collection_idUUIDDado bruto de origem
normalized_record_type / normalized_record_idString/UUIDRegistro normalizado gerado
analytics_record_type / analytics_record_idString/UUIDRegistro analítico gerado
collector_name / versionStringColetor que coletou
normalizer_name / versionStringNormalizer que processou
analytics_processor_name / versionStringProcessor que calculou
-- Rastrear origem de um produto normalizado
SELECT l.raw_collection_id, rc.source_name,
       rc.collected_at, l.normalizer_name
FROM data_lineage l
JOIN raw_collections rc ON l.raw_collection_id = rc.id
WHERE l.normalized_record_id = '<uuid-do-produto>';
schema_documentation
Documentação dos schemas raw e normalized

Armazena descrições dos campos de cada schema em fields_json, exemplos em examples_json e regras de validação em validation_rules_json.

entity_relationships
Grafo de relacionamentos entre entidades

Metadados do grafo ER do sistema. Cada linha representa uma aresta (source_entity → target_entity com relationship_type).

data_owners
Registro de responsáveis por módulo

Define quem é responsável técnico e de negócios por cada módulo. Usado para escalation e SLAs.

data_slas
Definições de SLA de frescor, disponibilidade e qualidade

Define os SLAs esperados por módulo/fonte: freshness_sla (intervalo máximo), availability_sla (% uptime), quality_sla (score mínimo).

data_contracts
Contratos de governança de dados
ColunaTipoDescrição
module / source_name / contract_nameStringIdentificação do contrato
criticalityStringlow | medium | high | critical
statusStringdraft | active | deprecated
raw_required / lineage_required / quality_requiredBooleanRequisitos do contrato
schema_rules_json / quality_rules_jsonJSONBRegras de schema e qualidade

Armazena respostas brutas das APIs para auditoria e re-parsing. Vinculado opcionalmente a um sportsbook.

Q Consultas Úteis

Saúde geral do pipeline

-- Taxa de sucesso por módulo nas últimas 24h
SELECT
    domain,
    count(*) filter (where status = 'success') sucesso,
    count(*) filter (where status = 'error')   erro,
    count(*) total,
    round(count(*) filter(where status='success')*100.0/count(*),1) pct_sucesso
FROM pipeline_runs
WHERE started_at >= now() - interval '24 hours'
GROUP BY domain ORDER BY domain;

Produtos pendentes de normalização

SELECT source_name, raw_schema_name, count(*) pendentes
FROM raw_collections
WHERE processing_status = 'normalization_pending'
  AND module = 'ecommerce'
GROUP BY source_name, raw_schema_name
ORDER BY pendentes DESC;

Histórico de preços de um produto

SELECT np.store_name, np.price, np.availability, np.collected_at
FROM normalized_products np
WHERE np.canonical_product_id = 'pampers-premium-care-m'
ORDER BY np.collected_at DESC
LIMIT 30;

Itens coletados hoje por loja

SELECT source_name, count(*) coletados, max(collected_at) ultima_coleta
FROM raw_collections
WHERE module = 'ecommerce'
  AND collected_at >= current_date
GROUP BY source_name ORDER BY coletados DESC;

Erros recentes de coletor

SELECT ce.collector_name, ce.error_type, ce.message,
       cr.started_at, ce.created_at
FROM collector_errors ce
JOIN collection_runs cr ON ce.run_id = cr.id
WHERE ce.resolved_at IS NULL
ORDER BY ce.created_at DESC LIMIT 20;

Lineagem de um produto (raw → normalizado)

SELECT
    rc.source_name,
    rc.collected_at,
    rc.raw_json->'scrapedProduct'->>'scraper_strategy' as estrategia,
    np.price,
    np.store_name,
    l.normalizer_name
FROM normalized_products np
JOIN data_lineage l   ON l.normalized_record_id = np.id::text
JOIN raw_collections rc ON rc.id = l.raw_collection_id
WHERE np.canonical_product_id = 'pampers-premium-care-m'
ORDER BY rc.collected_at DESC LIMIT 10;
